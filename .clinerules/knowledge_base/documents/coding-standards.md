# Coding Standards

## General Principles

- Prioritize readability and maintainability over cleverness.
- Keep functions and classes small and focused (Single Responsibility Principle).
- Use meaningful, descriptive names for variables, functions, and classes.
- Prefer explicit code over implicit magic.

## Naming Conventions

- Variables: camelCase (JavaScript/TypeScript) or snake_case (Python)
- Functions: verb + noun (e.g., `fetchUserProfile`, `calculateTotal`)
- Classes: PascalCase

## Code Style

- Maximum line length: 100 characters
- Use consistent indentation (2 or 4 spaces)
- Add comments only for complex logic or "why", not "what"
- Always handle errors gracefully with meaningful messages

## Testing

- Write unit tests for all new business logic
- Aim for high test coverage on critical paths
