# Coding Standards

## General Principles

- Prioritize readability and maintainability over cleverness.
- Keep functions, classes, and components small and focused.
- Use meaningful, descriptive names.
- Prefer explicit code over implicit behavior.

## Naming Conventions

- Variables and functions: camelCase (JavaScript/TypeScript)
- Classes and types: PascalCase
- Constants: UPPER_SNAKE_CASE
- Files: kebab-case (e.g., user-profile.service.ts)

## Code Style

- Maximum line length: 100 characters
- Consistent indentation (2 spaces preferred)
- Add comments only for "why", not "what"
- Always handle errors with meaningful messages
- Avoid deep nesting when possible

## Testing

- Write unit tests for all new business logic
- Aim for clear, readable tests
- Test both happy path and edge cases
